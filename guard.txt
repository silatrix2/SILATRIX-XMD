## 🔌 Plugin System

### Creating Plugins
Create file kwenye `plugins/` directory:

```javascript
// plugins/myPlugin.js
module.exports = {
  name: 'My Plugin',
  description: 'Plugin yangu',
  commands: ['hello', 'test'],
  
  async execute(sock, msg, args, context) {
    const { reply, sender, isOwner } = context;
    const command = args[0];
    
    if (command === 'hello') {
      await reply('Hello from my plugin!');
    }
  }
};
```

### Available Context
Plugin zako zinapata context hii:
- `sock` - WhatsApp socket connection
- `msg` - Message object
- `args` - Command arguments
- `sender` - Sender JID
- `isGroup` - Boolean, true ikiwa message ni group
- `isOwner` - Boolean, true ikiwa sender ni owner
- `config` - Bot configuration
- `reply(text)` - Function ya ku-reply
- `replyWithMention(text)` - Reply na mention

## 🚨 Troubleshooting

### Common Issues

1. **QR Code haionyeshi**
   - Check internet connection
   - Ensure port 5000 haijachukuliwa

2. **Plugin haziload**
   - Check file naming (must end with .js)
   - Check plugin syntax
   - Use `.reload` command

3. **Bot hajiconnect**
   - Delete session files na try again
   - Check WhatsApp account status

### Debugging
Enable debug mode:
```javascript
logger: P({ level: 'debug' })
```

## 📊 Performance Tips

1. **Optimize Plugins**
   - Use async/await properly
   - Handle errors gracefully

2. **Session Management**

3. **Message Handling**
